Detailed Description

Contains methods to perform multiple robot simple tasks.

This class contains methods that use the primitives tasks libraries in order to perform composed tasks. Examples of composed tasks which this class can perform are:

  • Ask a question to the human and waits for a human confirmation.
  • Find an object using the sensors and take it.
  • Move to a location and find a person. The main idea of this class is asociate each task with a ROS service to be available for other ROS nodes.

Performs the simple task to ask a question to the user and wait for a confirmation from the user.

This task consist of the following steps:

  1. The robot ask a question to the user.
  2. For a period of time, the robot waits for the user's answer (yes or not) and while the robot waits for the answer it will repeat the question with certain frequency.
Parameters
t_questionToAskThe question that the robot will ask to the user.
t_timeoutThe amount of time (milliseconds) of the robot's wait-for-answer.
t_repatTimeoutThe amount of time (milliseconds) that the robot must wait before repeat the question to the user if this does not respond. If no time given, the question will be repeated every 5000 millisecs.
Returns
True if the user answer with a positive confirmation to the robot's question. False otherwise.

Performs the task of ask for a person's name and store it.

Assuming there is a person in front of the robot. This tasks consists in the following steps:

  1. The robot asks for the person's name.
  2. The robot waits until the person provide its name. While the robot is waiting it will repeat the question with a specified frequency.
  3. When the person provides his name, the robot will ask for a name confimation to the person.
  4. If the confirmation is positive, the task end. If not, the task start again from the begginning (there will be an attempts limit).
Parameters
[out]t_personNameStores the obtained person's name.
[in]attemptTimeoutThe limit time of each learn name attempt.
[in]t_repeatTimeoutThe amount of time (milliseconds) that the robot must wait to ask to the user for her/his name again.
[in]t_maxTaskAttemptsThe max number of attempts in the confirmation step.
Returns
True if the person's name was obtained succesfully. False otherwise.

Perform the task of remeber a person's face.

This tasks consists in the following steps:

  1. There is a person in front of the robot (looking, or not, at the robot's camera).
  2. The robot start to instruct to the operator.
  3. After the instruction, the robot starts to search for a human face in front of it.
  4. If the robot does not find a face, it starts to move the head in different positions and tries to find a fece again.
  5. When the robot finds a face, it starts to training the face, at the end of this step the robot announce tht the training is finished.
Parameters
t_findInstructionsThe robot instructions for the human, during the find face phase. To speech the default instructions a parameter value of "default" must be passed.
t_faceNameThe name to assign to the remebered human.
t_headMovementsThe head movements to perform during the find search phase. To perform a default movements set, an empty list must be passed as parameter. The head movements consist of a vector of float pairs in which each pair indicates the headPan and headTilt respectively.
Returns
True if the human face was found and remember succesfully. False otherwise.

Perform the task of wait for a valid voice command comming from the user.

This task consist in the following steps:

  1. For a period of time, the robot wait's for a user voice command.
  2. When the robot hears a command, it try to parse it.
  3. If the parse is valid then, the command and the params are returned. If not, back to the step 1.
Parameters
[out]t_commandThe resulted parsed command. If no valid command is given during the valid period of time, the returned value will be an empty string.
[out]t_paramsThe resulted params of the parsed command.
[in]t_timeoutThe amount of time (milliseconds) that the robot will wait for a command.
